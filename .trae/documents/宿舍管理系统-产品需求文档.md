# 宿舍管理系统 Web 前端 - 产品需求文档 (PRD)

## 1. 产品概述
宿舍管理系统是一个面向高校和企业的住宿信息化管理平台，旨在简化宿舍资源管理、学生入住退宿流程、报修工单处理和费用核算等日常运营工作。
- 主要用户：系统管理员、学生、维修人员，解决传统手工管理效率低下、信息不透明的问题
- 目标价值：提升宿舍管理效率，降低运营成本，改善住宿体验，实现数据化决策

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 系统管理员 | 后台创建 | 全系统管理、数据导出、用户管理 |
| 学生 | 学号注册 | 个人信息查看、报修提交、费用缴纳 |
| 维修人员 | 后台创建 | 报修工单接收和处理 |

### 2.2 功能模块
1. **登录注册页**: 用户认证、角色选择、密码找回
2. **仪表盘首页**: 数据统计卡片、快捷操作入口、公告通知
3. **宿舍管理页**: 楼栋列表、房间详情、入住状态可视化
4. **学生管理页**: 学生信息列表、入住办理、退宿处理
5. **报修中心页**: 工单列表、状态筛选、详情查看
6. **费用管理页**: 费用账单、缴费记录、逾期提醒
7. **公告通知页**: 公告列表、发布管理

## 3. 核心流程

### 3.1 用户登录流程
用户访问系统 → 选择角色 → 输入用户名密码 → JWT认证 → 根据角色跳转对应首页

### 3.2 学生入住流程
管理员进入学生管理 → 点击办理入住 → 选择或新增学生 → 选择宿舍楼栋 → 选择房间 → 系统自动分配床位 → 确认入住

### 3.3 报修处理流程
学生提交报修 → 系统生成工单 → 维修人员接收工单 → 处理故障 → 更新状态 → 学生确认完成 → 反馈评价

## 4. 用户界面设计

### 4.1 设计风格
**设计理念**: 现代化办公风格 + 数据可视化导向，采用"卡片式布局 + 数据驱动"的设计语言

**色彩方案**:
- 主色调: #1E40AF (深蓝色 - 专业稳重)
- 辅助色: #3B82F6 (亮蓝色 - 活力科技)
- 成功色: #10B981 (绿色 - 入住完成)
- 警告色: #F59E0B (橙色 - 待处理提醒)
- 错误色: #EF4444 (红色 - 紧急工单)
- 背景: #F3F4F6 (浅灰色 - 清爽简洁)

**字体选择**:
- 标题字体: "PingFang SC" / "Microsoft YaHei" (中文)
- 正文字体: "Inter" / "系统默认" (易读性强)
- 数字字体: "DIN" / "Roboto Mono" (数据展示)

**布局风格**:
- 卡片式布局，圆角设计 (border-radius: 8px-12px)
- 顶部导航 + 侧边栏菜单
- 响应式网格系统 (Grid + Flexbox)
- 数据可视化优先 (图表、进度条、状态指示器)

**交互设计**:
- 平滑过渡动画 (transition: 0.3s ease)
- 表单即时验证反馈
- 悬停状态高亮
- 加载状态指示器
- 成功/失败提示消息

### 4.3 响应式设计
- **桌面端优先**: 宽屏设计，充分利用屏幕空间，多列布局
- **平板端适配**: 中等屏幕，保持核心功能，简化侧边栏
- **移动端适配**: 小屏幕，单列布局，底部导航，简化操作

### 4.4 组件设计规范
- **按钮**: 圆角设计，悬停阴影效果，加载动画
- **表单**: 标签浮动，验证提示，焦点高亮
- **表格**: 固定表头，行悬停高亮，排序指示器
- **卡片**: 统一圆角，阴影层次，边框强调
- **导航**: 固定顶部，图标+文字，活跃状态指示
- **模态框**: 居中显示，遮罩层，关闭按钮，动画过渡
